# Преобразование IR датасета в STS формат

## Проблема

IR (Information Retrieval) и STS (Semantic Textual Similarity) - это разные задачи:
- **IR**: Асимметричная задача (query → document), бинарная классификация (релевантно/нерелевантно)
- **STS**: Симметричная задача (sentence1 ↔ sentence2), регрессия (степень сходства 0-1)

## Решение: Преобразование IR → STS

### Стратегия преобразования

1. **Релевантные пары** (relevance_score > 0):
   - Если скор релевантности > 1: нормализуем (score / max_score)
   - Если скор = 1 (бинарный): присваиваем 0.8-1.0 (высокое сходство)
   - Добавляем небольшую вариацию для разнообразия

2. **Нерелевантные пары** (relevance_score = 0):
   - Присваиваем 0.0-0.2 (низкое сходство)
   - Добавляем вариацию для обучения

### Использование

В `train_cross_encoder_sts.py` добавьте в конфигурацию:

```python
{
    "name": "sberquad_ir_to_sts",
    "type": "ir_to_sts",
    "hf_repo": "kngrg/sberquad-retrieval",
    "hf_repo_qrels": "kngrg/sberquad-retrieval-qrels",
    "use_train": True,
    "use_test": False,
    "num_negatives": 1,  # Количество негативных примеров на позитивный
    "weight": 0.5,  # Вес для IR датасета (меньше чем STS)
}
```

### Преимущества

- ✅ Используем уже имеющиеся IR датасеты
- ✅ Увеличиваем объем обучающих данных
- ✅ Модель учится на реальных русских текстах

### Ограничения

- ⚠️ IR датасет не идеально подходит для STS (разные задачи)
- ⚠️ Преобразование может внести шум в данные
- ⚠️ Рекомендуется использовать с меньшим весом, чем чистые STS датасеты

## Рекомендации

1. **Используйте IR датасет с весом 0.3-0.5** (меньше чем STS)
2. **Комбинируйте с STS датасетами** для баланса
3. **Оцените качество** на тестовом наборе после обучения
