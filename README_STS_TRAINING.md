# Обучение Cross-Encoder для Semantic Textual Similarity (STS)

## Проблема с текущей моделью

Текущая модель (`models/cross-encoder-rusbeir`) обучена на **Information Retrieval (IR)** датасете (`sberquad-retrieval`), который предназначен для задачи **вопрос-ответ**. 

### Почему IR модель не подходит для аннотирования компетенций?

1. **Задача IR**: Найти релевантные документы для запроса
   - Модель учится различать: "релевантно" (1) vs "нерелевантно" (0)
   - Фокус на бинарной классификации релевантности

2. **Задача аннотирования компетенций**: Оценить семантическое сходство между компетенцией и сегментом текста
   - Нужна оценка **степени** сходства (0.0 - 1.0), а не просто "да/нет"
   - Важно различать: "очень похоже" (0.9) vs "немного похоже" (0.3)

### Результат использования IR модели:
- Модель дает высокие скоры (0.99) для многих сегментов, даже не очень релевантных
- Плохо различает степени сходства
- Не подходит для ранжирования по качеству соответствия

## Решение: Обучение на STS датасете

**Semantic Textual Similarity (STS)** - это задача оценки семантического сходства между двумя текстами.

### Преимущества STS обучения:

1. **Непрерывные скоры**: Модель учится предсказывать скоры от 0 до 1 (или 0 до 5)
2. **Градация сходства**: Модель различает разные степени сходства
3. **Подходит для ранжирования**: Можно сравнивать сегменты по качеству соответствия

### Датасет: STSb Multi MT

**Базовый вариант (только русский):**
- **Формат**: Пары предложений с оценками сходства (0-5)
- **Примеры**: 5749 для обучения, 1500 для dev, 1379 для тестирования
- **Пример**: 
  - `sentence1`: "Самолет взлетает."
  - `sentence2`: "Взлетает самолет."
  - `similarity_score`: 5.0 (полное сходство)

**Расширенный вариант (все языки):**
- **Всего примеров**: ~57,490 для обучения (10 языков × 5749)
- **Преимущество**: Увеличение датасета в 10 раз
- **Языки**: en, de, es, fr, it, nl, pl, pt, ru, zh
- **Примечание**: Мультиязычное обучение может улучшить обобщающую способность модели

## Обучение модели

### Запуск обучения:

```bash
python train_cross_encoder_sts.py
```

### Продолжение обучения (если прервалось):

```bash
python train_cross_encoder_sts.py --resume_from models/cross-encoder-sts/checkpoints/epoch_2
```

### Конфигурация:

- **Базовая модель**: `DeepPavlov/rubert-base-cased`
- **Эпохи**: 3
- **Batch size**: 16
- **Learning rate**: 2e-5
- **Выходная директория**: `models/cross-encoder-sts`

### Оценка качества:

Модель оценивается с помощью **корреляции Пирсона** между предсказанными и реальными скорами:
- **Pearson correlation**: Чем ближе к 1.0, тем лучше
- **Spearman correlation**: Чем ближе к 1.0, тем лучше

## Использование обученной модели

После обучения обновите `DEFAULT_CROSS_ENCODER_MODEL` в `annotation/annotator.py`:

```python
DEFAULT_CROSS_ENCODER_MODEL = "models/cross-encoder-sts/final_model"
```

## Ожидаемые результаты

После обучения на STS датасете модель должна:
1. ✅ Давать более точные скоры сходства (не только 0.99 или 0.00)
2. ✅ Лучше различать степени сходства между компетенциями и сегментами
3. ✅ Улучшить качество ранжирования сегментов внутри компетенций

## Время обучения

На RTX 4080 (12GB VRAM):
- **Базовый вариант** (только русский, ~7k примеров): ~1-2 часа для 3 эпох
- **Расширенный вариант** (все языки, ~57k примеров): ~8-12 часов для 3 эпох
- Зависит от batch size и других параметров

## Рекомендации по размеру датасета

**5749 примеров может быть недостаточно** для качественного обучения. Рекомендуется:

1. ✅ **Использовать все языки STSb Multi MT** (~57k примеров) - увеличит датасет в 10 раз
2. ✅ **Использовать dev split для обучения** (+1.5k примеров для русского)
3. ✅ **Transfer learning**: Начать с уже обученной IR модели (`models/cross-encoder-rusbeir/final_model`)

Текущая конфигурация в `train_cross_encoder_sts.py` уже настроена на использование всех языков и dev split.
