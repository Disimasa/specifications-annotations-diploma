# Мультиязычное обучение Cross-Encoder

## Вопрос: Испортит ли обучение на других языках качество?

**Короткий ответ**: Зависит от стратегии. Может помочь, может навредить.

## Факторы влияния

### 1. Объем данных на целевом языке

- **Мало данных на русском** (< 10k примеров):
  - ✅ Мультиязычное обучение **поможет** (transfer learning)
  - Модель учится общим паттернам семантического сходства
  - Другие языки дополняют недостающие примеры

- **Достаточно данных на русском** (> 20k примеров):
  - ⚠️ Мультиязычное обучение может **навредить**
  - Модель может "забыть" специфику русского языка
  - Другие языки создают шум

### 2. Близость языков

- **Славянские языки** (польский, чешский, украинский):
  - ✅ Близкая структура и лексика
  - ✅ Могут улучшить качество на русском
  - ✅ Рекомендуется использовать

- **Романские/Германские языки** (английский, немецкий, французский):
  - ⚠️ Дальше от русского
  - ⚠️ Могут ухудшить качество
  - ⚠️ Использовать с осторожностью

### 3. Стратегия обучения

#### Вариант 1: Взвешивание датасетов (рекомендуется)

```python
{
    "name": "stsb_multi_mt_ru",
    "language": "ru",
    "weight": 1.0,  # Полный вес для русского
},
{
    "name": "stsb_multi_mt_slavic",
    "languages": ["pl"],  # Польский
    "weight": 0.5,  # Половина веса
},
```

**Преимущества**:
- ✅ Больше примеров на русском
- ✅ Другие языки дополняют, но не доминируют
- ✅ Баланс между объемом данных и качеством

#### Вариант 2: Только славянские языки

```python
{
    "name": "stsb_multi_mt_slavic",
    "languages": ["pl", "cs", "uk"],  # Польский, чешский, украинский
    "weight": 0.7,  # Немного меньше чем русский
}
```

**Преимущества**:
- ✅ Близкие языки, меньше конфликтов
- ✅ Улучшение обобщающей способности
- ✅ Меньше риска ухудшения качества

#### Вариант 3: Все языки (не рекомендуется)

```python
{
    "name": "stsb_multi_mt_all",
    "languages": ["en", "de", "es", "fr", "it", "nl", "pl", "pt", "zh"],
    "weight": 0.3,  # Низкий вес
}
```

**Риски**:
- ⚠️ Много далеких языков
- ⚠️ Может ухудшить качество на русском
- ⚠️ Использовать только если данных на русском очень мало

## Рекомендации

### Для вашей задачи (аннотирование компетенций на русском):

1. **Базовый вариант** (только русский):
   - ~7k примеров (train + dev)
   - Быстрое обучение (~1-2 часа)
   - Хорошее качество на русском

2. **Рекомендуемый вариант** (русский + славянские):
   - ~7k русских + ~5.7k польских (weight=0.5) = ~10k примеров
   - Среднее время обучения (~2-3 часа)
   - Лучшая обобщающая способность

3. **Максимальный вариант** (все + IR):
   - ~7k русских + ~5k других (weight=0.3) + IR датасет = ~15k примеров
   - Долгое обучение (~4-6 часов)
   - Риск ухудшения качества на русском

## Эксперименты

Рекомендуется провести эксперименты:

1. Обучить модель только на русском
2. Обучить модель на русском + польском (weight=0.5)
3. Сравнить качество на тестовом наборе
4. Выбрать лучший вариант
