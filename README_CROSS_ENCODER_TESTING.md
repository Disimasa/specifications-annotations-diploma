# Тестирование Cross-Encoder моделей для аннотирования компетенций

## Контекст

В рамках улучшения качества аннотирования компетенций мы исследовали возможность использования cross-encoder моделей для re-ranking результатов от bi-encoder. Согласно подходу "retrieve-and-rerank" из статьи [Sentence Embeddings. Cross-encoders and Re-ranking](https://osanseviero.github.io/hackerllama/blog/posts/sentence_embeddings2/), cross-encoder может улучшить точность ранжирования после первоначального отбора кандидатов bi-encoder.

## Задача аннотирования компетенций

**Цель:** Определить, какие сегменты текста релевантны каждой компетенции из онтологии.

**Тип задачи:** Semantic Textual Similarity (STS) - оценка семантического сходства между текстами.

**Текущее решение:** Bi-encoder (`ai-forever/sbert_large_nlu_ru`) с косинусным сходством.

## Протестированные модели

### 1. `cross-encoder/ms-marco-MiniLM-L-12-v2` (английская)

**Тип:** MS-MARCO (вопрос-ответ, retrieval)

**Результаты тестирования:**
- Средний score на тестовых парах: 7.775
- Выдает raw logits в диапазоне примерно 7-8
- После нормализации через sigmoid все scores становятся ~0.999 (насыщение)
- После min-max нормализации различия между сегментами минимальны

**Проблемы:**
- Обучена на английских текстах, плохо работает с русскими
- Выдает очень похожие scores для всех сегментов (7.5-8.5)
- После нормализации различия исчезают (0.82-0.84, разница 0.025)

**Пример результатов:**
```
Raw scores: [7.44, 7.46, 7.78, 8.09, 8.38]
После sigmoid: [0.9994, 0.9994, 0.9996, 0.9997, 0.9998]  # Все почти одинаковые
После min-max: [0.0000, 0.0213, 0.3617, 0.6915, 1.0000]  # Лучше, но все равно плохо
```

### 2. `DiTy/cross-encoder-russian-msmarco` (русская)

**Тип:** MS-MARCO (вопрос-ответ, retrieval), обучена на русском MS-MARCO датасете

**Результаты тестирования:**
- Выдает нормализованные scores в диапазоне [0, 1]
- Хорошо различает релевантные и нерелевантные тексты в изолированных тестах
- НО: при использовании для аннотирования компетенций дает неправильные результаты

**Проблемы:**
- Обучена на задаче "вопрос-ответ", а не на семантическом сходстве
- Выдает высокие scores для нерелевантных сегментов ("ТЕХНИЧЕСКОЕ ЗАДАНИЕ" получает 0.51, что выше релевантных)
- Не различает степени релевантности для компетенций

**Пример результатов:**
```
Компетенция: "Литье по выплавляемым моделям"
Сегмент "ТЕХНИЧЕСКОЕ ЗАДАНИЕ": score = 0.5123
Сегмент "Разработка технологии литья...": score = 0.3926  # Релевантный, но score ниже!
```

### 3. `GrigoryT22/cross-encoder-ru` (русская)

**Тип:** RAG (вопрос-ответ), обучена для определения релевантности пар вопрос-ответ

**Результаты тестирования:**
- Выдает logits для бинарной классификации [не релевантно, релевантно]
- После softmax все probabilities становятся ~0.99+ для всех сегментов
- Даже пустой текст получает score 0.9980

**Проблемы:**
- Классификатор слишком "сильный" - сглаживает различия между эмбеддингами
- Вход в последний слой для всех пар практически одинаковый (norm=15.7062-15.7064)
- Модель делает бинарную классификацию, а не ранжирование по степени релевантности
- Не различает степени релевантности

**Детальный анализ:**
```
Пара: "Литье..." <-> "РАЗДЕЛ 1."
  Logits: [-3.9750, 3.5744], diff=7.5494
  Probability релевантно: 0.9995

Пара: "Литье..." <-> "" (пустой текст)
  Logits: [-3.9627, 3.5667], diff=7.5294
  Probability релевантно: 0.9995  # Даже пустой текст!

Пара: "Литье..." <-> "Разработка технологии литья..."
  Logits: [-4.2360, 3.7704], diff=8.0064
  Probability релевантно: 0.9997  # Релевантный, но почти такой же score
```

**Анализ классификатора:**
- Вход в последний слой для всех пар практически идентичен (norm≈15.7, mean≈0.001)
- Классификатор преобразует все разные эмбеддинги в очень похожие представления
- Веса последнего слоя: mean≈0.002, std≈0.035 (очень маленькие)
- Bias: [-0.0029, 0.0029] (минимальный)

## Выводы

### Почему cross-encoder модели не подходят:

1. **Неправильный тип задачи:**
   - Cross-encoder модели обучены на MS-MARCO (вопрос-ответ) или RAG
   - Наша задача - Semantic Textual Similarity (STS)
   - Модели определяют "релевантно/не релевантно", а не степень сходства

2. **Бинарная классификация vs ранжирование:**
   - Cross-encoder делает бинарную классификацию
   - Нам нужно ранжирование по степени релевантности
   - Bi-encoder с косинусным сходством лучше подходит для этой задачи

3. **Проблемы с нормализацией:**
   - Английские модели выдают raw logits, которые сложно нормализовать
   - Русские модели выдают нормализованные scores, но неправильно ранжируют

4. **Сглаживание различий:**
   - Классификатор в cross-encoder сглаживает различия между эмбеддингами
   - Все сегменты получают похожие scores, независимо от релевантности

### Рекомендации:

1. **Использовать только bi-encoder** (`rerank_top_k=0`) для русских текстов
   - Bi-encoder уже хорошо работает с русскими текстами
   - Косинусное сходство правильно различает степени релевантности
   - Не требует дополнительных вычислительных ресурсов

2. **Для улучшения качества в будущем:**
   - Обучить свой cross-encoder на STS датасете для русского языка
   - Или использовать подход Augmented SBERT для улучшения bi-encoder
   - Или собрать размеченные данные и fine-tune модель на своей задаче

## Технические детали

### Нормализация scores

**Проблема:** Cross-encoder модели выдают scores в разных форматах:
- Английские модели (ms-marco): raw logits (~7-8)
- Русские модели: нормализованные scores [0, 1] или logits

**Попытки нормализации:**
1. Sigmoid: `1 / (1 + exp(-x/5))` - приводит к насыщению для больших значений
2. Min-max: `(x - min) / (max - min)` - работает только внутри батча, искажает результаты
3. Прямое использование: для русских моделей, которые уже выдают нормализованные scores

**Вывод:** Нормализация не решает проблему, так как основная проблема - неправильный тип задачи.

### Архитектура моделей

**GrigoryT22/cross-encoder-ru:**
- Базовая модель: LaBSE (multilingual BERT)
- Классификатор: 5 полносвязных слоев с нормализацией
- Проблема: классификатор слишком "сильный" - сглаживает различия между эмбеддингами

**DiTy/cross-encoder-russian-msmarco:**
- Базовая модель: rubert-base-cased
- Обучена на русском MS-MARCO датасете
- Проблема: обучена на задаче "вопрос-ответ", а не на семантическом сходстве

## Статус

❌ **Cross-encoder re-ranking не рекомендуется** для задачи аннотирования компетенций на русских текстах.

✅ **Рекомендуется использовать только bi-encoder** (`rerank_top_k=0`).
